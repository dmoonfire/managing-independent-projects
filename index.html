Managing Independent Projects

Managing Independent Projects

Implementing ideas in 20 minutes a day

Dylan Moonfire

“I got this idea...”

Terminology

  • An idea is the desired end result
  • A project implements an idea
  • A project consists of many tasks
  • A session is a short time to do a task

Ideas

  • Has a well-defined goal
  • Can be achieved
  • Based on your passions and interests
  • Everyone's idea is different

Projects

  • Used to implement an idea
  • Requires time, money, and resources
  • Sufficiently complex to need planning

Independent Projects

  • Limited budget
  • Reduced staffing
  • Restricted time

Recipes for Failure

  • Technologies you have never used
  • Using skills that you never learned
  • Requires a horde of programmers and artists
  • Does not have well defined goals
  • Has 4 steps with step 3 being “magic happens”

Priorities

  • Consciously make priorities
  • Write them down
  • Use them to figure out schedules
  • Avoid guilt when not under pressure
  • Used to resist obsessions
“You have enough time to do anything, if you want it badly enough.”

Sessions

  • Time in front of the computer
  • Limited period of time to work
  • Based on your schedule
  • Working around your priorities

Realistic Planning

  • Schedule your distractions
  • Ensure your health
  • Work around your schedule
  • Know your limits

Making Sessions Productive

  • Reduce distractions
  • Prepare for the session
  • Do more with less time

Distractions

  • Television
  • Social Networking:
    • Facebook
    • Twitter
    • Instant Messaging
  • Browsing
  • Video games
  • Email

Preparing to Sit Down

  • How fast does your system boot?
  • IDE startup time
  • Schedule batch tasks:
    • Search indexes
    • Virus scanning
    • Defragment your drive
  • Source control check-ins and outs

In a Session

  • Work for the entire time
  • Focus on one thing at a time
  • Use remaining time for maintenance

After a Session

  • Figure out what is left
  • Determine new tasks
  • Prepare for the next one
  • Document the current state

Tasks

  • A single focused purpose
  • Takes a session to implement
  • Minimal impact to the code base

Identifying a Task

  • Work Breakdown Structures (WBS)
  • A list of things to do
  • Know how fast you can work
  • Always be willing to reorganize
Beyond the Session

Waterfall Method

  • Waterfall method is not popular for projects
  • Works well for a single task
  • Decisions slow down development
  • Planning reduce decisions while coding
  • Optimize the time to develop

Stealing Time

  • Commuting to work
  • Waiting in line
  • Doing chores
  • Watching TV
  • Doctor's office

Take Notes

  • Smart phone applications
  • Tomboy
  • Microsoft OneNote
  • Evernote
  • Notepad

Planning Techniques

  • Pseudo code
  • Flowcharts
  • Diagrams
  • GUI layouts

Pseudo Code

  • No Intellisense
    • Need to memorize code
    • Need to know the framework
  • No compiler
    • Need to code without errors
  • Limited buffer
    • 15-40 LOC in head
    • 100-200 LOC on paper
  • Easier to translate directly into code

Flowcharts

Development Techniques

Source Control

  • Multiple choices:
    • Git
    • Mercurial
    • Subversion
    • Team Foundation
  • Allows you to work on multiple machines
  • Prevents accidentally overwriting work
  • Can help you recover from failure
  • Branching is fantastic for testing ideas

Unit Tests

  • Test-Driven Development
  • Prevent regression bugs
  • Help bridge multiple-session tasks
  • Writing test generators
  • Core logic should be testable

Don't Reinvent the Wheel

  • Your time is precious
  • Other people figured it out already
  • You can't do everything

Reinventing it Anyways

  • Make it a conscious decision
  • Document reasons
  • Plan for the extended time

Education

  • Treat learning techniques as a task
  • Don't skimp on learning things
  • Don't use shiny new libraries just because

Code Cleanup

  • "Broken Windows"
  • Review the code
  • Get rid of the "masking tape"

Documentation

  • Document libraries
  • Create user manuals
  • Write help screens

Coding Standards

  • Easier to comprehend
  • Keep everything formatted
  • Faster to get to the right place
  • Simplify getting up to speed again

Picking Fights

  • Some tasks are easier later
  • Don't waste 5-10 sessions
  • Break it down even more
  • Path of Least Resistance

Steady Progress

  • A little bit every day adds up
  • Smooths out the rough days with good ones
  • Gives you a sense of progress

Project Lifecycle

  • Beginning
  • Middle
  • End
  • Aftermath
  • Failure

The Beginning

  • Everything is exciting!
  • Don't drop everything else
  • Plan, plan, and plan some more
  • Prepare for the project:
    • File layout
    • Project organization
    • Coding standards

The Middle

  • Hitting the hard parts
  • Tedious bugs and features
  • Hard to see the end
  • Don't change things to distract yourself

The Tedious Middle

  • Remind yourself of the goal
  • Avoid over-planning (feature creep)
  • Focus on one thing at a time
  • Keep working steadily

Additional Projects

  • Always worse in the middle
  • Stick to one project a time
  • Write good ideas down for later
  • Multiple projects can help burnout
    • ... but increase the time to do both.

Near the End

  • Going beyond “it works”
  • Make a list of what is remaining
  • Start getting feedback
  • Work on polish
  • Finish documentation and bug hunting

Aftermath

  • Take a break!
    • Play a game
    • Work on something else
  • You still aren't done

Failure

  • You will make mistakes
  • Don't take them seriously
  • “Fail Fast”
  • Be willing to change plans
  • Move to the next task

Support System

  • Family
  • Friends
  • Fans
  • Supporters
  • Patrons

Talking About You

  • Many different ways:
    • Blogs
    • Social networking
    • Hijacking co-workers
  • Good way to gather interest
  • Fans make you want to work
  • Talking too much can slow you down

When To Spend Money

  • As late as possible
  • Money means an investment
  • Turns it from a hobby into something serious
  • Costs change everything
  • Avoid committing yourself

Working With Others

  • Project management takes time
  • Differing opinions requires management
  • Two or more people speeds development
  • Requires different forms of investment
    • No one works for free
    • ... just not necessarily for money.
  • The increases the benefits and risks

Work

  • Don't compete with your boss
  • Don't steal from work
  • Do the work you are paid for
  • Don't using sick days

Friends and Family

  • Relationships take time and work
  • Support systems are critical
  • Time now will benefit you later
  • Keep your priorities clear

Yourself

  • Take care of yourself
  • Exercise
  • Take breaks
  • Reward yourself
